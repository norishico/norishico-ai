# ノリシコ競馬AI — Claude Code 運用ガイド

## プロジェクト概要

JRAの成績データ・払戻データをもとに馬券の期待値スコアを計算し、
買い条件を満たすレースの買い目を生成する競馬予想AIシステム。

## DBスキーマ（重要）

### results テーブル
- `horse_num` = スコアランク連番（実馬番ではない）
- `finish` = dividendsのumatan_uma1/uma2 + wideペアから逆引きで設定
- `horse_name` = 正確（照合済み）

### training テーブル（坂路・ウッドチップ調教データ）

| カラム名 | 内容 |
|---|---|
| `horse_name` | 馬名（resultsとTRIM照合でJOIN可能） |
| `date` | 調教日 |
| `trainer` | 調教師 |
| `venue` | 調教場所（美浦/栗東） |
| `source` | `sakuro`（坂路）or `woodc`（ウッドチップ） |
| `lap1` | 最終1Fタイム（92%有効・主要指標） |
| `lap2` | 最終2Fのうち前半1F（加速ラップ判定に使用） |
| `time3` / `time4` | 3F/4Fタイム（79%がNULL・現在未使用） |
| `sire` / `dam` | 父/母（2022年以降のみ） |
| `sex` / `age` | 性別/年齢 |

- 全150万件（坂路10.6万 / ウッドチップ139.9万）、2019-2026年
- resultsとのJOIN: `TRIM(r.horse_name) = t.horse_name` + レース前14日以内
- カバー率: 2022年以降82-93%、2019-21年は56-60%
- **加速ラップ判定**: `lap1 < lap2` → 仕上がり良好サイン（3着内率+3pt）
- **好調教判定**: 坂路 lap1<12.0 / WC lap1<11.5

### dividends テーブル（カラムマッピング確定・外部照合済み）

| カラム名 | 内容 |
|---|---|
| `tansho_payout` | 単勝払戻（100円あたり） |
| `umaren_payout` | 馬連払戻（100円あたり） |
| `wide1_payout` | ワイド1-2着ペア払戻 |
| `wide2_payout` | ワイド1-3着ペア払戻 |
| `wide3_payout` | ワイド2-3着ペア払戻 |
| `umatan_payout` | 馬単払戻 |
| `sanrenpuku_payout` | 3連複払戻 |
| `sanrentan_payout` | 3連単払戻 |
| `umatan_uma1` | 1着実馬番（信頼できる） |
| `umatan_uma2` | 2着実馬番（信頼できる） |

## 買い条件（v6.5 / 通常ゾーン+チャレンジゾーン+不良フィルタ+バイアスフィルタ）

### A. v6本体（スコアリング◎○ベース）

各クラスに「通常ゾーン」と「チャレンジゾーン」の2段階を設ける。
通常ゾーンは単勝+馬連の2,000円、チャレンジゾーンは単勝のみ1,000円。
**不良馬場は全クラス買い見送り**（ROI26%、予測不能な馬場変化のため）。

```python
def is_buy_v6(grade, heads, gap, odds, ev7, good_train=True, sire='', track_cond=''):
    """v6.5 通常ゾーン + チャレンジゾーン + 不良フィルタ
    Returns: ('normal'/'challenge', True) or (None, False)
    """
    if track_cond in ('不', '不良'): return None, False  # 不良馬場は買わない
    if odds is None or odds == 0: return None, False
    if not (2.0 <= ev7 <= 20.0): return None, False

    if grade == '新馬': return None, False
    if grade == '未勝利': return None, False

    if grade == '1勝':
        if not good_train: return None, False
        if 3 <= odds <= 5: return 'normal', True
        if 5 < odds <= 6: return 'challenge', True    # ROI141%, 4/6年黒字

    if grade == '2勝':
        # 【v6.6】2勝クラス廃止（2026-04-13検証で決定）
        # 7年BT: 単独赤字・的中率0-6%の構造的不振。廃止で全体+263,190円改善
        return None, False

    if grade == '3勝':
        if sire in SUNDAY_SIRES and not good_train: return None, False
        if not ((heads >= 12) or (gap >= 8)): return None, False
        if 8 <= odds <= 11 and (heads >= 12 or gap >= 8):  # v6.6: 8-11倍に絞り込み
            return 'normal', True
        if 20 < odds <= 25: return 'challenge', True   # ROI97%, 4/7年黒字

    if grade == 'G3':
        if 3 <= odds <= 16 and 3 <= gap <= 8 and heads >= 14:
            return 'normal', True

    if grade in ('G1', 'G2'):
        if 5 <= odds <= 20: return 'challenge', True   # ROI167%(G2)/129%(G1)

    return None, False
```

#### 通常ゾーン vs チャレンジゾーン

| クラス | 通常ゾーン（★★/★★★） | チャレンジゾーン（★） |
|---|---|---|
| **1勝** | **廃止(v6.6)** (細帯0.5倍幅は実運用で買えない) | **廃止(v6.6)** |
| **2勝** | **廃止(v6.6)** | **廃止(v6.6)** |
| 3勝 | odds **8-11**（heads12+ or gap8+）【v6.6で細帯分析により絞り込み】 | **odds 20-25**（ROI97%） |
| **G3** | **廃止(v6.6)** (小サンプルで構造的不振) | なし |
| **G1/G2** | なし | **odds 5-20**（ROI167%） |

- 通常ゾーン: 単勝◎1,000円 + 馬連◎○1,000円 = 2,000円
- チャレンジゾーン: 単勝◎1,000円のみ = 1,000円
- チャレンジの根拠: 上限超えは馬連ROIが低いが単勝ROIは黒字維持
- G1/G2の根拠: odds5-20倍の中穴◎はROI167%(G2)/129%(G1)。市場が過小評価する中穴帯

#### 調教フィルタ
- **1勝クラス**: `has_good_train=True` の馬のみ買い
- **3勝クラス**: SS系で非好調教の馬を除外
- `has_good_train`: 坂路 lap1<12.0 / WC lap1<11.5

#### 弱セグメント: 廃止
- ROI 77.3%で構造的赤字のため廃止

#### 不良馬場フィルタ（v6.5で追加）
- `track_cond in ('不', '不良')` → 全クラス買い見送り
- 根拠: 不良馬場ROI 26.1%（38R/-48,750円）、1番人気勝率30.8%
- 稍重(ROI105.9%)・重(ROI117.4%)は良馬場より高ROIのため買い対象のまま
- 道悪スコアリング補正（score_course_fitnessの40%ブレンド）は無効のまま維持（稍重・重で既に黒字のため変更不要）

#### 強バイアス不利枠フィルタ（v6.5で追加）
- 買い判定後、honmeiが以下3パターンの内枠（馬番/頭数≤0.35）なら見送り
- スコアリングは変更しない（honmei選定に影響なし、買い判定のみフィルタ）

| # | パターン | バイアス方向 | 不利枠ROI差 | 根拠 |
|---|---|---|---|---|
| 1 | 新潟芝・後半（wn>5）× 内枠 | 外有利 | **-68.6pt** | 直線長く外差し圧倒的有利 |
| 2 | 札幌芝・後半（wn>5）× 内枠 | 外有利 | **-63.1pt** | 洋芝の傷みで内側走路悪化 |
| 3 | 中山ダ・前半（wn≤3）× 内枠 | 外有利 | **-53.1pt** | 砂被り+スタート不利 |

- バックテスト結果: 7年中5年改善、ROI+1.7pt、損益+30,050円
- 除外17R中16Rがハズレ（的中回避率94%）
- スコアペナルティ方式は8回連続悪化のため不採用、フィルタ方式を採用

### B. 別枠ルール（調教×血統パターン / スコアリング不要）

```python
KINGKAME_SIRES = frozenset({
    'キングカメハメハ','ロードカナロア','ルーラーシップ','ドレフォン','レイデオロ',
    'ホッコータルマエ','ロゴタイプ',
})

def is_special_buy(grade, odds, popularity, heads, accel, good_train, sire):
    """別枠買いルール: C2(新馬) + F1(未勝利)"""
    if not accel: return False, ''
    sire_s = (sire or '').strip()
    is_mainstream = sire_s in SUNDAY_SIRES or sire_s in KINGKAME_SIRES

    # C2: 新馬×非主流血統×中穴×加速ラップ×多頭数
    if not is_mainstream:
        if grade == '新馬' and 10 <= odds < 20 and heads >= 15:
            return True, 'C2_新馬accel'

    # F1: 未勝利×主流血統(SS+KK)×穴馬×好調教+加速ラップ
    if is_mainstream and good_train:
        pop = popularity or 99
        if grade == '未勝利' and 20 <= odds < 30 and 1 <= pop <= 8:
            return True, 'F1_未勝利主流accel'

    return False, ''
```

#### C2: 新馬×非主流血統×加速ラップ
- 対象: **全出走馬**（スコアリング◎○に依存しない）
- 条件: 新馬×非主流血統×odds10-20×加速ラップ(lap1<lap2)×15頭以上
- **1レース最大1頭**（lap1最速を優先）
- 買い目: **単勝1,000円のみ**（ワイドはROI低下要因のため廃止）
- 根拠: 非主流血統は市場が過小評価するが、加速ラップが出ていれば仕上がりは本物。多頭数で紛れが出やすい（5/7年黒字、ROI122%）
- ワイド廃止の根拠: ワイド的中時の平均回収2,630円（投資2,000円に対して微益）、ハズレ時の追加損失900円がROIを圧迫

#### F1: 未勝利×主流血統×好調教+加速ラップ
- 対象: **全出走馬**
- 条件: 未勝利×主流血統(SS系+KK系)×odds20-30×好調教+加速ラップ×1-8番人気
- **1レース最大1頭**（lap1最速を優先）
- 買い目: **単勝1,000円のみ**
- 根拠: 主流血統で未勝利のまま人気が落ちた馬は市場が過小評価。好調教+加速ラップなら覚醒のタイミング（4/6年黒字、ROI175%）
- v6本体が不調の年にプラスを出す傾向（ポートフォリオ効果）

## 買い目（v6.5 / 3段階推奨）

### 買い判定フロー

```
1. is_buy_v6() で通常ゾーン？
   → ◎オッズ 8倍以上: 単勝500円 + 馬連1,500円 = 2,000円（馬連寄せ）
   → ◎オッズ 8倍未満: 単勝1,000円 + 馬連1,000円 = 2,000円（標準）
2. is_buy_v6() でチャレンジゾーン？ → ★ 単勝のみ 1,000円
3. is_special_buy() で別枠C2/F1？ → ★ 単勝のみ 1,000円
4. いずれも非該当 → 軸馬のみ提示（買い推奨なし）
```

### 買い目配分（v6.5 あかり案A）

| 枠 | 条件 | 買い目 | 投資/R |
|---|---|---|---|
| **v6通常(高オッズ)** | normal & odds≥8 | 単勝◎500円 + 馬連◎○1,500円 | 2,000円 |
| **v6通常(低オッズ)** | normal & odds<8 | 単勝◎1,000円 + 馬連◎○1,000円 | 2,000円 |
| **v6チャレンジ** | challenge | 単勝◎1,000円のみ | 1,000円 |
| **別枠C2** | C2_新馬accel | 単勝1,000円のみ | 1,000円 |
| **別枠F1** | F1_未勝利主流accel | 単勝1,000円のみ | 1,000円 |

- 高オッズ帯で馬連比率を上げる根拠: 高オッズ◎の馬連ROIが低オッズ帯より高い（的中時の回収大）

## 会場×距離×父 ボーナス（venue_sire_bonus）

スコアリングの最終段階で、血統と特定コースの相性が全体平均より+12pt以上高いパターンにボーナスを加算。

### 仕組み
- `venue_sire_bonus`テーブル: 会場×距離×父ごとの複勝率乖離とボーナス値を格納
- `build_venue_sire_bonus.py`で構築（cutoff_date以前のデータのみ使用、リーク防止）
- `calc_venue_sire_bonus(venue, dist, sire, conn)` → 0〜5pt加算
- ボーナス値 = min(max(乖離pt × 0.15, 1.5), 5.0)

### 構築条件
- サンプル30件以上
- 全体平均（sire×surface×距離帯）との乖離+12pt以上
- 年度安定性50%以上（3年以上データがあり、過半数の年で複勝率25%以上）

### 効果
- v6本体ROI: 132.4% → **135.6%**（+3.2pt）
- 2022年の赤字幅66%縮小、2026年が赤字→黒字転換
- 7年中5年で改善、1年横ばい、1年微減

## 🚨 必須ルール（エラー防止）

1. **馬名・着順・払戻を提示する際は必ず外部データ（netkeiba等）で照合してから回答する**
2. 推測で馬名を書かない。DBの値を `SELECT` で直接出力してから確認する
3. `horse_num` は実馬番ではなくスコアランク。着順判定には `finish` を使う
4. `finish` の確認には `umatan_uma1/uma2` と dividends を照合する
5. 文字化けした外部ページから馬名を読み取らない

## 🚨 実運用現実性ルール（バックテストの罠防止）

**バックテストは最終確定オッズで判定するため、「完璧に狭帯に入った馬だけ買えた」理想ケースを示す。実運用ではオッズ変動で買えない可能性を常に考慮すること。**

1. **オッズ幅が1倍未満の買い条件は不採用**（例: 4.0-4.5倍）
   - 理由: 発走時オッズの変動は平均±0.3-0.5倍、1倍幅未満だと3回に2回は圏外
2. **バックテストで高ROIでも、バンド幅が狭ければ実効ROI = BT × 捕捉率**
   - 捕捉率 ≒ 1 - (平均変動 ÷ バンド幅)
3. **1勝/2勝の狭帯中心タイプは構造的に実運用不可**
   - 上記2クラスの現行ルールは「完全廃止」
4. **採用ルールのバンド幅目安**
   - normal買い: **最低 2倍幅以上**（例: 8-11倍, 5-7倍）
   - チャレンジ: **最低 3倍幅以上**（例: 5-10倍）
5. **auto_refresh の ±20%ロック機構は必須**
   - 一度買い判定された馬は、±20%以内のオッズ変動で判定維持
   - これにより朝判定→発走までの軽微な変動を許容
